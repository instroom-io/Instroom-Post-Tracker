# Instroom Post Tracker

Multi-tenant B2B SaaS for influencer marketing agencies. Automatically detects, downloads, and tracks influencer posts across Instagram, TikTok, and YouTube — with frozen 7-day metrics, EMV calculation, and Google Drive archiving.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 15 (App Router) + TypeScript strict |
| Database | Supabase (Postgres + Auth + RLS) |
| Styling | Tailwind CSS with custom design tokens |
| Client state | Zustand + TanStack Query v5 |
| Post ingestion | EnsembleData polling API |
| File storage | Google Drive via service account |
| Deployment | Vercel (cron jobs) |

## Architecture

```
Agency (super admin)
  └── Brand Workspaces
        └── Campaigns
              └── Influencers + Posts
```

- **Post detection**: Cron job polls EnsembleData every 30 minutes per active campaign influencer
- **Downloads**: Separate cron job processes a retry queue every 5 minutes, uploading watermark-free media to Google Drive
- **Metrics**: Cron job fetches 7-day-frozen performance data every 10 minutes
- **Workspace creation**: Brands onboard via a unique link (`/onboard/[token]`) generated by the agency

## Local Development

### Prerequisites
- Node.js 20+
- Supabase project (free tier works)
- EnsembleData account with API key
- Google Cloud service account with Drive API access

### Setup

```bash
cp .env.local.example .env.local   # fill in all values
npm install
npm run db:push                     # apply schema to Supabase
npm run db:generate                 # regenerate TypeScript types
npm run dev                         # http://localhost:3000
```

Create a test workspace (dev only): log in at `/login`, then navigate to `/onboarding`.

## Environment Variables

See `.env.local.example` for all required variables with setup instructions.

| Variable | Required | Notes |
|---|---|---|
| `NEXT_PUBLIC_SUPABASE_URL` | Yes | Supabase project URL |
| `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` | Yes | Supabase anon key |
| `SUPABASE_SERVICE_ROLE_KEY` | Yes | **Never expose to browser** |
| `NEXT_PUBLIC_APP_URL` | Yes | `http://localhost:3000` locally; production URL in production |
| `ENSEMBLE_API_KEY` | Yes | EnsembleData token |
| `ENSEMBLE_API_URL` | No | Defaults to `https://ensembledata.com/apis` |
| `ENSEMBLE_WEBHOOK_SECRET` | No | Reserved for future use |
| `GOOGLE_SERVICE_ACCOUNT_JSON_B64` | Yes | `cat key.json \| base64 \| tr -d '\n'` |
| `CRON_SECRET` | Yes | `openssl rand -hex 32` |

## Cron Jobs

Defined in `vercel.json`. All routes authenticate via `Authorization: Bearer <CRON_SECRET>`.

| Route | Purpose |
|---|---|
| `/api/cron/posts-worker` | Scrape new posts from EnsembleData |
| `/api/cron/download-worker` | Download media to Google Drive |
| `/api/cron/metrics-worker` | Fetch 7-day performance metrics |

Manual trigger: `curl -H "Authorization: Bearer <SECRET>" https://<URL>/api/cron/posts-worker`
